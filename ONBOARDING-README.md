# trinket-gcr — untracked onboarding files

These files are **not in git** (they're gitignored) but are needed to run the app
locally. After cloning the repo, extract this archive **from the repo root** so the
paths line up:

```bash
git clone <repo-url> trinket-gcr
cd trinket-gcr
git checkout gcr-firebase      # the branch you'll be working on
tar xzf trinket-onboarding.tar.gz
npm install                     # or however deps are installed for this project
```

## ⚠️ SENSITIVE — contains live secrets

This archive includes **real credentials and private keys**. Share it only over a
secure channel (not email/Slack/cloud links). It contains:

- `.env` — `SESSION_PASSWORD`, `GOOGLE_CLIENT_SECRET`, `LTI_PRIVATE_KEY`, project/billing IDs
- `config/local.yaml` — session cookie password, Firebase client config (apiKey)
- `config/firebase-service-account.json` — **GCP service-account private key**
- `lti-private-key.pem` — **LTI 1.3 RSA private key**

Consider rotating these or issuing the new dev their own credentials rather than
sharing yours long-term.

## Contents

| Path | Purpose |
|------|---------|
| `.env` | Environment variables (secrets + GCP project config) |
| `config/local.yaml` | Local app config (db backend, buckets, auth, LTI authority) |
| `config/firebase-service-account.json` | GCP service-account key for Firestore/Storage |
| `lti-private-key.pem` | LTI 1.3 tool signing key |
| `public/components/angular-scrollfix.js` | Vendored Angular component the app loads |
| `public/components/angular-slugify.js` | Vendored Angular component the app loads |
| `scripts/test-*.js` (31) | Ad-hoc LTI dev/test helper scripts (run against live Firestore) |

## Intentionally NOT included (regenerated or transient)

- `config/runtime.json` (+ `*.tmp-*`) — regenerated by the app on boot
- `firestore-debug.log` — emulator log
- `.superpowers/sdd/*`, `.claude/settings.local.json`, `*.bak` — local tooling/work artifacts
